about

This site was generated with Pelican, a static-site generator powered by Python and is hosted on a Raspberry Pi 2. It also relies on the following open source projects: Prism, -prefix-free, jQuery, three.js, MathaJax, Hyphenator, Lato and STIX.

Apart from extensive use of JavaScript; CSS3 and HTML5 is also used. The flexbox and 3D transformation properties are probably the least support in current and old browsers. Further, calc() is also used, but the support should be good enough in modern browsers. Hyphens is a CSS3/CSS4 property that isn’t widely supported, and therefore a JavaScript library is used instead. If JavaScript is enabled, HTML5 is utilised to manipulate the browser history and simulate a single page app.

This site was designed on a Mac, and primarily targets Chrome. This means that layout variations might occur across other systems. However, the site should work fine in all modern dekstop and mobile browsers; something updated in late 2014. On Windows, it has been tested without issue in Internet Explorer 10+ (although the page animation is broken until IE supports transform-style: preserve-3d;).

[2014-08-30] The low performance of overflow: scroll;, as it is implemented today (especially on mobile devices), forces me to use the default overflow: visible; property. This means that the scrollTop value is shared between all pages which in turn causes choppy animations if the current scrollTop value is higher than it can ever be on the new page. A solution would be to force the user to navigate strictly from the top, but it’s simply not an option. I’m (slowly) working on a smarter solution.

[2014-09-14] Following the general advice of coding the drop down (horizontal) menu in JavaScript instead of using the checkbox hack, a fallback had to be implemented. When users have JavaScript disabled the menu is shown as is, expanded. This can create problems on devices with smaller screens in portrait mode due to no document reflow. I am slowly looking into this. Other than that, the :hover selector is a problem on mobile devices, causing the menu icon not to transform back to the logotype when pressed to collapse the menu. It returns to its default value after the user presses anywhere else on the document.
